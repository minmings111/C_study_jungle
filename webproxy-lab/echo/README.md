# Echo Example

이 폴더는 CS:APP 스타일의 간단한 echo 서버/클라이언트 예제입니다.  
이 폴더만 따로 봐도 빌드와 실행이 가능하도록 구성되어 있습니다.

## 들어 있는 파일

- `echoserveri.c`
  - 서버 프로그램의 시작점입니다.
  - 포트를 열고 클라이언트 연결을 기다립니다.
- `echo.c`
  - 서버가 받은 문자열을 그대로 다시 보내는 핵심 로직입니다.
- `echoclient.c`
  - 클라이언트 프로그램의 시작점입니다.
  - 서버에 연결하고 문자열을 보내고 응답을 출력합니다.
- `csapp.h`, `csapp.c`
  - 소켓과 입출력 관련 도우미 함수들이 들어 있습니다.
- `Makefile`
  - 이 폴더 전용 빌드 파일입니다.
- `echo_study.md`
  - 파일별 읽는 순서와 함수 흐름 중심 학습 문서입니다.
- `socket_study.md`
  - 소켓, 파일 디스크립터, `Rio_*` 함수, 책임 구분 중심 학습 문서입니다.

## 빌드 방법

이 폴더로 이동한 뒤 아래 명령을 실행하세요.

```bash
make
```

빌드가 끝나면 아래 실행 파일이 만들어집니다.

- `echoserveri`
- `echoclient`

정리하려면:

```bash
make clean
```

## 실행 방법

먼저 서버를 실행합니다.

```bash
./echoserveri 8080
```

다른 터미널을 하나 더 열고 클라이언트를 실행합니다.

```bash
./echoclient 127.0.0.1 8080
```

그다음 클라이언트 터미널에서 문자열을 입력하고 엔터를 누르면,
서버가 같은 문자열을 그대로 다시 보내 줍니다.

예:

```text
hello
```

## 실행 흐름

1. 서버가 포트 `8080`에서 연결을 기다립니다.
2. 클라이언트가 `127.0.0.1:8080`으로 연결합니다.
3. 클라이언트가 문자열을 보냅니다.
4. 서버의 `echo()`가 그 문자열을 읽습니다.
5. 서버가 같은 문자열을 다시 보냅니다.
6. 클라이언트가 응답을 출력합니다.

## 디버깅할 때 먼저 볼 곳

- 서버 시작: `echoserveri.c` 의 `main`
- 연결 수락: `Accept(...)`
- echo 처리: `echo.c` 의 `echo(int connfd)`
- 클라이언트 전송: `echoclient.c` 의 `Rio_writen(...)`
- 클라이언트 응답 읽기: `echoclient.c` 의 `Rio_readlineb(...)`

## 핵심 개념 한 줄 정리

- `listenfd`: 서버가 연결을 기다리는 소켓 번호
- `connfd`: 서버가 클라이언트와 실제 통신하는 소켓 번호
- `clientfd`: 클라이언트가 서버와 통신하는 소켓 번호
- `buf`: 실제 문자열 데이터를 저장하는 배열
- `Rio_*`: 소켓으로 읽고 쓰기 쉽게 만든 도우미 함수
